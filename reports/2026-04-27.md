# Claude Code ベスプラ調査レポート 2026-04-27

## 今週の新発見

### モデル設定: `opusplan` エイリアスとハイブリッドモデルワークフロー

**発見内容**: Claude Code に `opusplan` という特殊モデルエイリアスが存在する。Plan Mode 中は Opus（高度な推論）、実行フェーズでは自動的に Sonnet（高速・低コスト）に切り替わる。

**モデルエイリアス一覧**（公式ドキュメント確認済み）:

| エイリアス | 動作 |
|-----------|------|
| `best` | 最高性能モデル（現在は `opus` と同等） |
| `opus` | 複雑な推論タスク向け最新 Opus |
| `sonnet` | 日常的なコーディング向け最新 Sonnet |
| `haiku` | 高速・軽量タスク向け |
| `opus[1m]` | Opus + 100万トークンコンテキスト |
| `sonnet[1m]` | Sonnet + 100万トークンコンテキスト |
| **`opusplan`** | **Plan Mode → Opus / 実行 → Sonnet の自動切替** |

**コスト最適化の効果**: 計画フェーズのみ Opus レートを支払い、実装フェーズは Sonnet レートで済む。大規模リファクタリング・機能設計・アーキテクチャ検討に特に有効。

**適用提案**:
```bash
# ~/.claude/settings.json
{
  "model": "opusplan"
}

# または起動時に指定
claude --model opusplan

# セッション中に切り替え
/model opusplan
```

### モデル設定: `CLAUDE_CODE_SUBAGENT_MODEL` によるサブエージェントコスト制御

**発見内容**: サブエージェントが使用するモデルを個別制御できる環境変数 `CLAUDE_CODE_SUBAGENT_MODEL` が存在。メインセッションは Opus で高度推論を行いつつ、サブエージェントの単純タスクは Sonnet や Haiku で処理することでコストを大幅削減できる。

**関連環境変数**:

| 環境変数 | 説明 |
|----------|------|
| `CLAUDE_CODE_SUBAGENT_MODEL` | サブエージェント専用モデル |
| `ANTHROPIC_DEFAULT_OPUS_MODEL` | `opus` エイリアスの解決先モデル ID |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | `sonnet` エイリアスの解決先 |
| `ANTHROPIC_DEFAULT_HAIKU_MODEL` | `haiku` エイリアスの解決先（旧 `ANTHROPIC_SMALL_FAST_MODEL`） |
| `ANTHROPIC_CUSTOM_MODEL_OPTION` | `/model` ピッカーにカスタムエントリを追加 |

**適用提案**:
```bash
# ~/.claude/settings.json の env セクション
{
  "env": {
    "CLAUDE_CODE_SUBAGENT_MODEL": "haiku",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-haiku-4-5-20251001"
  }
}
# → メインは Opus で推論、探索・調査サブエージェントは Haiku で 80% コスト削減
```

### エフォートレベル: Opus 4.7 の 5 段階制御と `xhigh` デフォルト化

**発見内容**: Opus 4.7 は他モデルにない `xhigh` と `max` を含む 5 段階のエフォートレベルをサポート。v2.1.117 以降、Opus 4.7 のデフォルトは `xhigh` に変更された。

**エフォートレベル対応表**:

| レベル | Opus 4.7 | Opus 4.6 / Sonnet 4.6 | 使いどころ |
|--------|----------|----------------------|-----------|
| `low` | ✓ | ✓ | レイテンシ重視の単純タスク |
| `medium` | ✓ | ✓ | コスト削減・品質妥協可能な作業 |
| `high` | ✓ | ✓ | 知能重視タスクの最低推奨値 |
| `xhigh` | ✓ | ✗ | コーディング・エージェントタスクの推奨デフォルト |
| `max` | ✓ | ✓ | 最高品質（セッション限定・過思考注意） |

**重要な動作仕様**:
- `max` はセッション限定（`CLAUDE_CODE_EFFORT_LEVEL` 経由では永続化）
- スキル・サブエージェントのフロントマターで `effort: xhigh` を個別指定可能
- `/effort auto` でモデルデフォルトにリセット
- Opus 4.6 で `xhigh` を設定すると `high` にフォールバック

**適用提案**:
```json
// ~/.claude/settings.json
{
  "effortLevel": "xhigh"
}
```
```bash
# 単発の深い推論はプロンプトで指示（effortLevel を変えない）
"ultrathink を使ってこのアーキテクチャを分析してください"
```
```yaml
# .claude/skills/review/SKILL.md フロントマター
---
effort: max  # レビューのみ最高品質、セッションレベルは変更しない
---
```

### settings.json: `permissions.ask` による確認プロンプト制御

**発見内容**: `permissions` に `allow`・`deny` に加えて `ask`（確認プロンプト）という第三のティアが存在することが公式ドキュメントで明確化された。リスクのある操作を完全ブロックせず、毎回確認を求められる。

**3 ティア構造**:
```json
{
  "permissions": {
    "allow": [
      "Bash(git diff *)",
      "Bash(npm test)"
    ],
    "ask": [
      "Bash(git push *)",
      "Bash(git reset *)",
      "Write(*.config.*)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Write(.env*)",
      "WebFetch"
    ]
  }
}
```

**設計方針**:
- `allow`: 完全自動（確認なし）
- `ask`: 毎回確認ダイアログを表示
- `deny`: 完全ブロック（理由を Claude に伝達）
- リストにない操作はデフォルトモード（`defaultMode` 設定）で処理

**適用提案**: `git push` や設定ファイル変更のような「やり直しが難しい操作」を `ask` に移行し、完全ブロックより柔軟な安全網を構築する。

### settings.json: 新設定項目（2026 年追加分）

**発見内容**: v2.1.100 以降に複数の有用な設定が追加された。

**新設定項目**:

```json
{
  "showThinkingSummaries": true,
  "awaySummaryEnabled": true,
  "fastModePerSessionOptIn": true,
  "disableAutoMode": "disable",
  "autoScrollEnabled": false
}
```

| 設定 | デフォルト | 説明 |
|------|-----------|------|
| `showThinkingSummaries` | false | 拡張思考の要約をインタラクティブセッションに表示 |
| `awaySummaryEnabled` | true | 数分離席後に戻った際の1行セッション要約を表示 |
| `fastModePerSessionOptIn` | false | Fast Mode をセッション限定にして永続化を防ぐ |
| `disableAutoMode` | 未設定 | `"disable"` で auto パーミッションモードを無効化 |
| `autoScrollEnabled` | true | フルスクリーンレンダリングで最新出力を自動スクロール |

**配列設定のマージ動作**（重要）:
```json
// ユーザー設定 ~/.claude/settings.json
{ "sandbox.filesystem.allowWrite": ["/opt/tools"] }

// プロジェクト設定 .claude/settings.json
{ "sandbox.filesystem.allowWrite": ["~/.kube"] }

// 結果: 両方が有効（置き換えではなく連結・重複除去）
```
`permissions.allow/deny/ask` や `availableModels` も同様にスコープ間でマージされる。

**適用提案**:
```json
// ~/.claude/settings.json
{
  "showThinkingSummaries": true,
  "awaySummaryEnabled": true,
  "fastModePerSessionOptIn": true,
  "permissions": {
    "ask": ["Bash(git push *)"]
  }
}
```

### プロンプトキャッシュ: モデル別の細粒度制御

**発見内容**: プロンプトキャッシュをモデル種別ごとに個別 ON/OFF できる環境変数が存在。デバッグ時や特定クラウドプロバイダーとの相性問題の解決に使える。

```bash
# 全モデルのキャッシュを無効化
DISABLE_PROMPT_CACHING=1

# モデル別に無効化
DISABLE_PROMPT_CACHING_HAIKU=1   # Haiku のみ
DISABLE_PROMPT_CACHING_SONNET=1  # Sonnet のみ
DISABLE_PROMPT_CACHING_OPUS=1    # Opus のみ
```

**使用場面**:
- キャッシュが原因で古いコンテキストを掴んでいる疑いがある場合
- Bedrock/Vertex AI でキャッシュ実装の違いによる問題が発生している場合
- コスト分析でキャッシュヒット率を測定する際の A/B 比較

**適用提案**:
```json
// デバッグ用 .claude/settings.local.json
{
  "env": {
    "DISABLE_PROMPT_CACHING": "1"
  }
}
```

### 5 つのエージェントワークフローパターン（体系化）

**発見内容**: Claude Code のエージェント活用パターンが公式・コミュニティ双方で 5 種類に体系化された。タスクの性質に応じて選択する指針が明確化されている。

| パターン | 概要 | 適用場面 |
|---------|------|---------|
| **Sequential** | ステップを順番に実行 | シンプルなタスク、デバッグしやすい |
| **Operator** | オーケストレーターが専門サブエージェントに委譲 | 複雑タスクの並列化・専門化 |
| **Split-and-Merge** | 独立したサブタスクを並列実行して結果をマージ | 50関数のドキュメント生成等 |
| **Agent Teams** | 複数Claudeインスタンスが共有タスクリストで協調（実験的） | 大規模・長期プロジェクト |
| **Headless** | 非インタラクティブ自動実行 | CI/CD、バッチ処理（十分テスト後のみ） |

**Split-and-Merge の具体例**:
```bash
# 50個の関数を10バッチに分割して並列ドキュメント生成
# 3〜4倍の高速化が報告されている
ls src/**/*.ts | xargs -P 10 -I{} \
  claude -p "このファイルの全関数にJSDocを追加: {}" \
  --allowedTools "Read,Edit"
```

**Operator パターンの推奨構成**:
```markdown
<!-- .claude/agents/orchestrator.md -->
---
name: orchestrator
description: 複雑なタスクをサブエージェントに委譲するオーケストレーター
model: opus
---
タスクを分析し、適切なサブエージェント（explore/implement/review）に分割して割り当てる。
```

**適用提案**: 現在手動で順番に実行しているリサーチ→実装→レビューの流れを Operator パターンで自動化する。

### MCP: HTTP トランスポートへの移行と自動再接続

**発見内容**: SSE トランスポートが非推奨となり、HTTP トランスポート（OAuth 2.1 対応）が 2026 年の標準となった。また MCP サーバーの切断時には指数バックオフで自動再接続される。

**トランスポート種別**:
| 種類 | 状態 | 認証 |
|------|------|------|
| `stdio` | 現役（ローカル） | - |
| `http` | **推奨**（リモート） | OAuth 2.1 |
| `sse` | **非推奨** | - |

**自動再接続動作**（HTTP/SSE）:
- 最大 5 回試行
- 遅延: 1s → 2s → 4s → 8s → 16s（指数バックオフ）
- 再接続中は `/mcp` で `pending` 状態として表示

**適用提案**:
```json
// .mcp.json - HTTP トランスポートへ移行
{
  "mcpServers": {
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp/",
      "headers": {
        "Authorization": "Bearer ${GITHUB_TOKEN}"
      }
    }
  }
}
```
