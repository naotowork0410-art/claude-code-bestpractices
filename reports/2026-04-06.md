# Claude Code ベスプラ調査レポート 2026-04-06

## 今週の新発見

### パーミッションモード（Permission Modes）

**発見内容**: Claude Code には5種類のパーミッションモードがある。特に `auto` モード（v2.1.83+）が新登場。`Shift+Tab` でセッション中にサイクル切り替え可能。

| モード | 自動許可される操作 | 主なユースケース |
|--------|-------------------|----------------|
| `default` | 読み取りのみ | 通常作業・センシティブな操作 |
| `acceptEdits` | 読み取り＋ファイル編集 | コードレビューしながら編集 |
| `plan` | 読み取りのみ（変更なし） | コードベース調査・計画立案 |
| `auto` | 全操作（バックグラウンド安全チェック付き） | 長時間タスク・承認疲れ解消 |
| `dontAsk` | 事前許可済みツールのみ | CI・スクリプト完全非対話 |
| `bypassPermissions` | 保護パス以外すべて | コンテナ・VM 内の隔離環境のみ |

**auto モードの注意点**:
- Team / Enterprise / API プランのみ（Pro/Max 不可）
- Claude Sonnet 4.6 / Opus 4.6 のみ（Haiku 不可）
- デフォルトでブロックされる操作: `curl | bash`、本番デプロイ、IAM 権限変更、`main` への直接 push、ファイルの大量削除など

**保護パス（全モードで自動承認されない）**:
- `.git`, `.vscode`, `.idea`, `.husky`
- `.gitconfig`, `.bashrc`, `.zshrc`, `.mcp.json`, `.claude.json`
- `.claude/` 配下（ただし `.claude/commands`, `.claude/agents`, `.claude/skills`, `.claude/worktrees` は除く）

**適用提案**:
```json
// .claude/settings.json
{
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}
```
```bash
# auto モードを有効化（Team/Enterprise/API プラン必要）
claude --enable-auto-mode

# CI で完全非対話実行（許可済みツールのみ）
claude --permission-mode dontAsk -p "run lint and tests"
```

### 新しい Hooks イベントと Hook タイプ

**発見内容**: 前回調査（18種類）から大幅に拡充。新規イベントと新しい Hook タイプが追加。

**新規 Hook イベント（前回レポートにない）**:

| イベント | 用途 |
|---------|------|
| `CwdChanged` | ディレクトリ変更時（direnv 連携に有用） |
| `FileChanged` | 特定ファイルの変更監視 |
| `ConfigChange` | 設定ファイル変更の監査・ブロック |
| `PermissionRequest` | 許可ダイアログをフックで自動応答 |
| `PermissionDenied` | auto モード classifier がブロックした時 |
| `PreCompact` / `PostCompact` | コンパクション前後 |
| `InstructionsLoaded` | CLAUDE.md 読み込み時 |
| `SubagentStart` / `SubagentStop` | サブエージェントの起動・停止 |
| `TaskCreated` / `TaskCompleted` | エージェントチームのタスク管理 |
| `TeammateIdle` | エージェントチームメンバーがアイドル状態に |
| `WorktreeCreate` / `WorktreeRemove` | worktree の作成・削除 |
| `Elicitation` / `ElicitationResult` | MCP サーバーがユーザー入力を要求 |
| `SessionEnd` | セッション終了時（理由別マッチャー対応） |
| `UserPromptSubmit` | プロンプト送信直後（Claude 処理前） |
| `StopFailure` | API エラーによるターン終了 |
| `PostToolUseFailure` | ツール呼び出し失敗後 |

**新しい Hook タイプ**:

1. **`prompt` 型** — LLM（Haiku デフォルト）が判断する宣言的フック
```json
{
  "hooks": {
    "Stop": [{
      "hooks": [{
        "type": "prompt",
        "prompt": "全タスクが完了しているか確認。未完なら {\"ok\": false, \"reason\": \"残作業内容\"} を返す。"
      }]
    }]
  }
}
```

2. **`agent` 型** — ファイル読み込み・コマンド実行ができるエージェントフック
```json
{
  "hooks": {
    "Stop": [{
      "hooks": [{
        "type": "agent",
        "prompt": "テストスイートを実行して全テストがパスすることを確認してください。",
        "timeout": 120
      }]
    }]
  }
}
```

3. **`http` 型** — HTTP エンドポイントへ POST
```json
{
  "hooks": {
    "PostToolUse": [{
      "hooks": [{
        "type": "http",
        "url": "http://localhost:8080/hooks/tool-use",
        "headers": { "Authorization": "Bearer $MY_TOKEN" },
        "allowedEnvVars": ["MY_TOKEN"]
      }]
    }]
  }
}
```

**`if` フィールド（v2.1.85+）** — ツール名＋引数で絞り込み（`matcher` はツール名のみ）:
```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "if": "Bash(git *)",
        "command": ".claude/hooks/check-git-policy.sh"
      }]
    }]
  }
}
```

**適用提案**:
```json
// direnv 連携: ディレクトリ変更時に環境変数を自動リロード
{
  "hooks": {
    "CwdChanged": [{
      "hooks": [{ "type": "command", "command": "direnv export bash >> \"$CLAUDE_ENV_FILE\"" }]
    }],
    "FileChanged": [{
      "matcher": ".envrc|.env",
      "hooks": [{ "type": "command", "command": "direnv export bash >> \"$CLAUDE_ENV_FILE\"" }]
    }]
  }
}
```

```json
// PermissionRequest フックで ExitPlanMode を自動承認
{
  "hooks": {
    "PermissionRequest": [{
      "matcher": "ExitPlanMode",
      "hooks": [{
        "type": "command",
        "command": "echo '{\"hookSpecificOutput\": {\"hookEventName\": \"PermissionRequest\", \"decision\": {\"behavior\": \"allow\"}}}'"
      }]
    }]
  }
}
```

```json
// ConfigChange: 設定ファイル変更を監査ログに記録
{
  "hooks": {
    "ConfigChange": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "jq -c '{timestamp: now | todate, source: .source, file: .file_path}' >> ~/claude-config-audit.log"
      }]
    }]
  }
}
```

### チェックポイント・リワインド（Checkpointing & /rewind）

**発見内容**: Claude Code はすべてのファイル編集を自動的にチェックポイントとして記録。`Esc+Esc` または `/rewind` でリワインドメニューが開き、以下から選択可能:

- **コードと会話を復元**: 両方を指定時点に戻す
- **会話のみ復元**: コードはそのままで会話を巻き戻す
- **コードのみ復元**: 会話はそのままでファイルを巻き戻す
- **ここから要約**: 指定メッセージ以降を圧縮してコンテキストを節約

チェックポイントはセッション間をまたいで保存（デフォルト30日）。外部プロセスによる変更は追跡されない点に注意。

**セッションのフォーク**:
```bash
# 現在のセッションを保持したまま別ブランチで試行
claude --continue --fork-session
```

**適用提案**:
- 大規模リファクタリング前に `/rewind` でチェックポイントを確認
- 実験的な変更は気軽に試せる（失敗したら `Esc+Esc` で即リワインド）
- 「ここから要約」で長いデバッグセッションの後半だけを圧縮（前半の重要指示は保持）

### プラグインシステム（Plugin System）

**発見内容**: スキル・エージェント・hooks・MCP サーバーを `.claude-plugin/plugin.json` にまとめて配布・共有できる**プラグイン**が追加。

**スタンドアロン（`.claude/`）とプラグインの使い分け**:
| アプローチ | スキル名 | 適したケース |
|-----------|---------|------------|
| スタンドアロン | `/hello` | 個人ワークフロー・単一プロジェクト |
| プラグイン | `/my-plugin:hello` | チーム共有・複数プロジェクト横断・コミュニティ配布 |

**プラグイン構造**:
```
my-plugin/
├── .claude-plugin/
│   └── plugin.json        # マニフェスト（必須）
├── skills/                # スキル（SKILL.md）
├── agents/                # カスタムエージェント定義
├── hooks/
│   └── hooks.json         # フック設定
├── .mcp.json              # MCP サーバー設定
├── .lsp.json              # LSP サーバー（コードインテリジェンス）
├── bin/                   # PATH に追加される実行ファイル
└── settings.json          # デフォルト設定（agent キーのみ対応）
```

```json
// .claude-plugin/plugin.json
{
  "name": "my-plugin",
  "description": "チーム共通ワークフロー集",
  "version": "1.0.0",
  "author": { "name": "Your Team" }
}
```

**適用提案**:
```bash
# 開発中のプラグインをテスト
claude --plugin-dir ./my-plugin

# 複数プラグインを同時ロード
claude --plugin-dir ./plugin-a --plugin-dir ./plugin-b

# プラグインの変更を反映（再起動不要）
/reload-plugins
```

既存の `.claude/` 設定をプラグインに変換する手順:
```bash
mkdir -p my-plugin/.claude-plugin
cp -r .claude/skills my-plugin/
cp -r .claude/agents my-plugin/
# hooks は settings.json から my-plugin/hooks/hooks.json へ移植
```

### エージェントチーム（Agent Teams）【実験的機能】

**発見内容**: 複数の Claude Code インスタンスをチームとして協調動作させる**エージェントチーム**が追加（実験的）。サブエージェントとの違いは「チームメート同士が直接メッセージを送り合える」点。

**有効化**:
```json
// settings.json
{
  "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" }
}
```

**サブエージェント vs エージェントチーム**:
| 項目 | サブエージェント | エージェントチーム |
|------|----------------|-----------------|
| コミュニケーション | 結果を親に返すのみ | チームメート同士で直接通信 |
| 調整方式 | 親エージェントが管理 | 共有タスクリストで自律調整 |
| 最適用途 | 結果のみ必要な集中タスク | 議論・協調が必要な複雑作業 |
| トークンコスト | 低い | 高い（各チームメートが独立コンテキスト） |

**推奨パターン**:
```text
# 並列コードレビュー
Create an agent team to review PR #142. Spawn three reviewers:
- One focused on security implications
- One checking performance impact
- One validating test coverage

# 競合仮説による調査
Spawn 5 agent teammates to investigate different hypotheses about [bug].
Have them talk to each other to disprove each other's theories.
```

**ベストプラクティス**:
- チームサイズ: 3〜5人が推奨（コスト・調整オーバーヘッドのバランス）
- タスク数: チームメート1人あたり5〜6タスクが最適
- 表示モード: `in-process`（Shift+Down で切り替え）または `split-pane`（tmux/iTerm2）
- 設定: `~/.claude.json` の `teammateMode` で変更可能
- チームメートは同一ファイルを編集しないよう作業を分割する

**エージェントチームフック**:
```json
// タスク完了前にテスト通過を強制
{
  "hooks": {
    "TaskCompleted": [{
      "hooks": [{
        "type": "agent",
        "prompt": "タスクのコードが全テストをパスしているか確認してください。"
      }]
    }]
  }
}
```

### セッション管理・CLI オプションの強化

**発見内容**: セッション再開・非対話実行・分岐など、運用に役立つ CLI オプションが充実。

**セッション再開**:
```bash
claude --continue           # 最近のセッションを再開
claude --resume             # 最近のセッション一覧から選択
claude --continue --fork-session  # 現在のセッションを保持してフォーク
```

**セッション名付け**: `/rename` コマンドで `"oauth-migration"` `"debugging-memory-leak"` などの名前をつけてブランチのように管理。

**非対話モード（CI/スクリプト用）**:
```bash
# ワンオフ実行
claude -p "Explain what this project does"

# 構造化出力（スクリプト連携）
claude -p "List all API endpoints" --output-format json

# ストリーミング JSON
claude -p "Analyze this log file" --output-format stream-json

# パイプライン統合
claude -p "<prompt>" --output-format json | your_command

# ツール制限付きバッチ処理（安全な並列実行）
for file in $(cat files.txt); do
  claude -p "Migrate $file from React to Vue" \
    --allowedTools "Edit,Bash(git commit *)" &
done
wait
```

**適用提案**:
- 長期タスク（1週間続くリファクタリング等）は `/rename` で名前をつけて管理
- CI パイプラインへの組み込みは `--output-format json` + `--allowedTools` の組み合わせ
- `--verbose` は開発中のみ使用し本番では外す

### Explore→Plan→Implement→Commit 4 フェーズワークフロー

**発見内容**: 公式が推奨する 4 フェーズワークフロー。Plan Mode を使って実装前の調査・計画を分離することで、間違った問題を解く失敗を防ぐ。

```
フェーズ1【Explore】: Plan Mode でコードを読み、変更なしで質問
  └─ "read /src/auth and understand how we handle sessions"

フェーズ2【Plan】: Plan Mode で詳細な実装計画を作成
  └─ "I want to add Google OAuth. What files need to change? Create a plan."
  └─ Ctrl+G でテキストエディタで計画を直接編集可能

フェーズ3【Implement】: Normal Mode に切り替えてコーディング
  └─ "implement the OAuth flow from your plan. write tests and fix failures."

フェーズ4【Commit】: コミット・PR 作成
  └─ "commit with a descriptive message and open a PR"
```

**適用提案**:
- 単一ファイルの軽微な変更はフェーズ1〜2をスキップ（オーバーヘッドになる）
- 複数ファイルをまたぐ変更・不慣れなコードは必ずフェーズ1〜2を実行
- `Ctrl+G` で計画をエディタで編集してから実行する習慣をつける

### CLAUDE.local.md（個人プロジェクト設定）

**発見内容**: プロジェクトルートに `CLAUDE.local.md` を配置すると、git 管理なしの個人専用プロジェクト設定として機能する（チームの `CLAUDE.md` と分離）。

| ファイル | 用途 | git 管理 |
|---------|------|---------|
| `~/.claude/CLAUDE.md` | 全プロジェクト共通の個人設定 | なし |
| `./CLAUDE.md` | プロジェクトのチーム設定 | あり（共有） |
| `./CLAUDE.local.md` | プロジェクトの個人設定 | なし（.gitignore 推奨） |

**適用提案**:
```bash
# .gitignore に追加
echo "CLAUDE.local.md" >> .gitignore
```
```markdown
# CLAUDE.local.md の例
# このプロジェクトでの私個人の注意事項
- テストは `bun test` ではなく `pnpm test` を使う（私の環境設定）
- DB マイグレーションは自動実行しないこと（本番DBに接続中）
- API キーは .env.local に設定済み
```

### LLMインタビューパターン（AskUserQuestion を活用した要件定義）

**発見内容**: 大きな機能実装前に Claude に「インタビュアー」役を担わせ、考慮不足を洗い出す公式推奨パターン。

**使い方**:
```text
I want to build [brief description]. Interview me in detail using the AskUserQuestion tool.

Ask about technical implementation, UI/UX, edge cases, concerns, and tradeoffs.
Don't ask obvious questions, dig into the hard parts I might not have considered.

Keep interviewing until we've covered everything, then write a complete spec to SPEC.md.
```

その後、SPEC.md ができたら **新しいセッション**で実装を開始（クリーンなコンテキスト）。

**適用提案**:
- `.claude/skills/spec-writer/SKILL.md` として定型化する:
```yaml
---
name: spec-writer
description: 機能仕様をインタビュー形式で作成する
disable-model-invocation: true
---
$ARGUMENTS の実装について、AskUserQuestion ツールを使って詳細なインタビューを実施してください。
技術実装、UI/UX、エッジケース、トレードオフを深掘りしてください。
インタビュー完了後、SPEC.md に完全な仕様書を作成してください。
```

### `/loop` コマンドとスケジュールタスク（新機能）

**発見内容**: セッション内で定期実行するプロンプトをスケジュールできる `/loop` コマンドが追加。

```
/loop 5m check if the deployment finished
/loop check the build every 2 hours
/loop 20m /review-pr 1234        # 別スキルを定期実行
remind me at 3pm to push the release branch   # 一度だけのリマインダー
```

- 間隔省略時のデフォルト: 10分
- セッションあたり最大50タスク
- 繰り返しタスクは7日で自動期限切れ
- `CLAUDE_CODE_DISABLE_CRON=1` で無効化

**デスクトップタスク**（クラウドタスクより低機能だがローカルファイルにアクセス可能）も存在。クラウドタスクは Anthropic インフラで実行し永続化するが、ローカルファイルは不可。

**適用提案**:
```
/loop 10m check if CI on the current PR has completed
/loop 1h summarize new GitHub issues and add them to TODO.md
```

### Channels — 外部メッセージアプリ連携（新機能 Q1 2026）

**発見内容**: Telegram / Discord / iMessage（macOS のみ）から実行中の Claude Code セッションに外部イベントをプッシュできる **Channels** 機能が追加。

Remote Control（claude.ai からローカルセッションを操作する機能）とは異なり、Channels は外部システムから Claude Code にメッセージを「プッシュイン」する仕組み。

**セットアップ例（Telegram）**:
```bash
/plugin install telegram@claude-plugins-official
/telegram:configure <bot-token>
claude --channels plugin:telegram@claude-plugins-official
# ペアリング: /telegram:access pair <code>
# アクセス制限: /telegram:access policy allowlist
```

**主なユースケース**:
- スマートフォンからコードが実行中のマシンに指示を送る
- CI 失敗 → Webhook → ファイルが開いている Claude Code セッションへ通知
- Claude が同じチャンネルで返信可能

### Git Worktree 分離（`--worktree` フラグ）

**発見内容**: `--worktree` フラグで自動的に git worktree を作成し、分離された環境で作業できる。

```bash
claude --worktree feature-auth    # .claude/worktrees/feature-auth/ + 新ブランチを作成
claude --worktree                  # 自動命名（例: "bright-running-fox"）
claude --worktree feature-auth --tmux  # tmux セッションで起動
```

**`.worktreeinclude` ファイル**（プロジェクトルートに配置）: gitignore 形式で、新しい worktree にコピーすべき gitignored ファイルを指定。
```
# .worktreeinclude
.env
.env.local
config/secrets.json
```

**適用提案**:
```bash
# .gitignore に worktrees ディレクトリを追加
echo ".claude/worktrees/" >> .gitignore
# .worktreeinclude を作成
echo ".env\n.env.local" > .worktreeinclude
```

### Sandbox 機能（ファイルシステム・ネットワーク分離）

**発見内容**: Bash コマンドの実行環境をサンドボックス化できる新設定が追加。ファイルシステムアクセスとネットワークアクセスを細かく制御可能。

```json
// .claude/settings.json
{
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": true,
    "excludedCommands": ["docker *"],
    "filesystem": {
      "allowWrite": ["/tmp/build", "~/.kube"],
      "denyRead": ["~/.aws/credentials"]
    },
    "network": {
      "allowedDomains": ["github.com", "*.npmjs.org"],
      "allowUnixSockets": ["/var/run/docker.sock"],
      "allowLocalBinding": true
    }
  }
}
```

### settings.json 新設定項目（大幅拡充）

**発見内容**: 前回レポートに未収録の重要設定項目が多数追加されている。

**モデル・推論制御**:
```json
{
  "model": "claude-opus-4-6",
  "effortLevel": "high",          // "low" / "medium" / "high"（Opus/Sonnet 4.6 のみ）
  "alwaysThinkingEnabled": true,  // 拡張思考をデフォルト有効化
  "showThinkingSummaries": true   // 思考ブロック全文表示（デフォルトは折りたたみ）
}
```

**UI カスタマイズ**:
```json
{
  "language": "japanese",          // Claude の応答言語を固定
  "spinnerTipsEnabled": false,     // スピナーTipsを無効化
  "spinnerTipsOverride": {
    "excludeDefault": true,
    "tips": ["Use /loop for recurring tasks"]
  },
  "prefersReducedMotion": true,    // UI アニメーション無効化
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh"
  }
}
```

**Git/Attribution**:
```json
{
  "attribution": {
    "commit": "Generated with AI\n\nCo-Authored-By: AI <ai@example.com>",
    "pr": ""    // 空文字でPRへの帰属を無効化
  },
  "includeGitInstructions": false  // 組み込みのgitワークフロー指示を削除（カスタムスキル使用時）
}
```

**アップデートチャンネル**:
```json
{
  "autoUpdatesChannel": "stable"   // デフォルトは "latest"（1週間遅れでリグレッション回避）
}
```

**グローバル設定 `~/.claude.json`**（settings.json ではなく）:
```json
{
  "autoConnectIde": true,        // 外部ターミナルから起動中 IDE に自動接続
  "editorMode": "vim",           // 入力プロンプトに Vim キーバインド
  "showTurnDuration": false,     // "Cooked for 1m 6s" メッセージを非表示
  "teammateMode": "in-process"   // エージェントチーム表示モード
}
```

### LSP プラグイン（コードインテリジェンス）

**発見内容**: Language Server Protocol（LSP）対応のプラグインにより、Claude がコードの定義ジャンプ・参照検索・診断情報を取得可能に。

**速度差**: テキスト検索 45 秒 → LSP ナビゲーション 50 ミリ秒

**公式 LSP プラグインが対応する言語**: TypeScript, Python, Go, Rust, Java, C/C++, C#, PHP, Kotlin, Ruby, HTML/CSS

使用できる操作: `goToDefinition`, `findReferences`, `hover`, `documentSymbol`, `getDiagnostics`

**適用提案**:
```bash
/plugin install typescript@claude-plugins-official
/plugin install python@claude-plugins-official
```
