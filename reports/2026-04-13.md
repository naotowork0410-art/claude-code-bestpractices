# Claude Code ベスプラ調査レポート 2026-04-13

## 今週の新発見

### Skills フロントマター完全リファレンス

**発見内容**: `SKILL.md` のフロントマターには多数のフィールドがあり、特に以下が実用的（前回レポートの基本情報から大幅拡充）。

| フィールド | 説明 |
|-----------|------|
| `argument-hint` | オートコンプリート時に表示されるヒント例: `[issue-number]` |
| `user-invocable: false` | メニュー非表示にしつつ Claude には自動呼び出しを許可（バックグラウンド知識として使用） |
| `allowed-tools` | このスキル実行中に承認なしで使えるツール（スペース区切り or YAML リスト） |
| `effort` | 推論量: `low` / `medium` / `high` / `max`（Opus 4.6 のみ） |
| `context: fork` | 独立したサブエージェントコンテキストで実行 |
| `agent` | `context: fork` 時に使用するエージェント型（`Explore` / `Plan` / カスタム） |
| `hooks` | このスキルのライフサイクルにスコープされた hooks |
| `paths` | ファイルパターンに一致するときのみ自動読み込み（例: `"*.py,src/**"` ） |
| `shell` | `bash`（デフォルト）または `powershell` |

**スキル呼び出し制御の全パターン**:

| フロントマター設定 | ユーザー呼び出し | Claude 自動呼び出し | コンテキスト読み込み |
|---|---|---|---|
| デフォルト | ✅ | ✅ | description が常時ロード |
| `disable-model-invocation: true` | ✅ | ❌ | description は非ロード |
| `user-invocable: false` | ❌ | ✅ | description が常時ロード |

**適用提案**:
```yaml
# ~/.claude/skills/deploy/SKILL.md
---
name: deploy
description: Deploy the application to production
disable-model-invocation: true   # 手動のみ：自動デプロイ防止
allowed-tools: Bash(git *) Bash(npm run build)
effort: high
---
デプロイ手順...
```

```yaml
# .claude/skills/api-conventions/SKILL.md  
---
name: api-conventions
description: REST API design conventions for our services
user-invocable: false    # Claude が自動適用、メニュー非表示
paths: "src/api/**,*.controller.ts"   # API ファイル編集時のみ自動ロード
---
API 規約...
```

### Skills の引数インデックスアクセスと変数展開

**発見内容**: `$ARGUMENTS` 以外に個別引数にアクセスする構文と、セッション変数が利用可能。

| 変数 | 説明 |
|------|------|
| `$ARGUMENTS` | 全引数（空白区切り文字列） |
| `$ARGUMENTS[N]` または `$N` | N番目の引数（0始まり） |
| `${CLAUDE_SESSION_ID}` | 現在のセッション ID（ログ・ファイル命名に活用） |
| `${CLAUDE_SKILL_DIR}` | このスキルが格納されているディレクトリの絶対パス |

**適用提案**:
```yaml
# ~/.claude/skills/migrate-component/SKILL.md
---
name: migrate-component
description: Migrate a component from one framework to another
argument-hint: "[component-name] [from-framework] [to-framework]"
---

$0 コンポーネントを $1 から $2 に移行してください。
既存の動作とテストをすべて保持してください。

# ログを記録
作業ログを logs/${CLAUDE_SESSION_ID}.md に保存してください。
```
実行例: `/migrate-component SearchBar React Vue`

### Skills のシェルインジェクション（動的コンテキスト注入）

**発見内容**: `` !`command` `` 構文でシェルコマンドの実行結果を Claude に渡す前に展開できる（前回レポートの `!` プレフィックスの詳細化）。

```yaml
# ~/.claude/skills/pr-summary/SKILL.md
---
name: pr-summary
description: Summarize changes in a pull request
context: fork
agent: Explore
allowed-tools: Bash(gh *)
---

## PR コンテキスト
- diff: !`gh pr diff`
- コメント: !`gh pr view --comments`
- 変更ファイル: !`gh pr diff --name-only`

## タスク
このプルリクエストを日本語で要約してください。
```

複数行版（fenced block）:
````markdown
## 環境情報
```!
node --version
npm --version
git status --short
```
````

**セキュリティ設定**: 組織ポリシーとして無効化できる。
```json
// .claude/settings.json（マネージド設定に推奨）
{
  "disableSkillShellExecution": true
}
```

**適用提案**: CI 情報・git log・環境変数などをスキル実行時に自動注入してコンテキストを強化する。`${CLAUDE_SKILL_DIR}` でスキルバンドルのスクリプトを参照。

### Skills のサポートファイルパターン

**発見内容**: スキルディレクトリには複数ファイルを配置でき、必要な時だけロードされる。SKILL.md は 500 行以内を推奨。

```
~/.claude/skills/my-skill/
├── SKILL.md           # メイン（必須）
├── reference.md       # 詳細 API ドキュメント（必要時のみロード）
├── examples.md        # 使用例
└── scripts/
    └── helper.py      # Claude が実行するスクリプト
```

```yaml
# SKILL.md からサポートファイルを参照
## リソース
- 完全な API 詳細は [reference.md](reference.md) を参照
- 使用例は [examples.md](examples.md) を参照
```

**適用提案**: 大規模な API ドキュメントや規約は `reference.md` に分離し、SKILL.md は概要と Navigation に集中させる。スキル説明は 250 文字以内で、重要なキーワードを冒頭に置く（超過分は切り捨てられる）。

### コンテキスト管理の高度テクニック

**発見内容**: `/compact` と `/rewind` の高度な使い方が公式ドキュメントで明確化。

1. **ターゲットコンパクション**: 汎用コンパクションより具体的に
   ```bash
   /compact Focus on the API changes and test failures
   ```

2. **CLAUDE.md にコンパクション指示を追加**: 重要情報をコンパクション後も保持
   ```markdown
   # CLAUDE.md
   ## コンパクション指示
   コンパクション時は常に以下を保持すること:
   - 変更したファイルの完全なリスト
   - テストコマンドとその結果
   - 未解決の TODO リスト
   ```

3. **部分的な要約（Summarize from here）**: `Esc+Esc` → チェックポイント選択 → "Summarize from here" で特定時点以降のみ要約

4. **セッション命名 `/rename`**: セッションを後から検索しやすくする
   ```bash
   /rename oauth-migration
   /rename debugging-memory-leak
   ```

5. **コンテキスト劣化の目安**: パフォーマンスは 20〜40% 使用率から劣化開始、60% 超で指示の見落としが頻発 → ステータスラインでモニタリング推奨

**適用提案**:
```json
// ~/.claude/settings.json
{
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "65"
  }
}
```
ステータスラインにコンテキスト使用率を表示し、60% 超前に `/clear` するワークフローを確立。

### CLI ツール統合と非インタラクティブモード

**発見内容**: Claude は CLI ツール（`gh`、`aws`、`gcloud`、`sentry-cli` 等）を natively 活用できる。また非インタラクティブ出力フォーマットが拡充。

**CLI ツール活用**:
```bash
# gh CLI がない場合、API 呼び出しのレートリミットに当たりやすい
# gh CLI をインストールして Claude に活用させる
claude -p "gh CLI を使って open な PR をすべてリストし、レビュー待ち時間が長いものから並べてください"
```

**未知の CLI ツールも学習させる**:
```
foo-cli-tool --help を使って foo ツールを理解し、A・B・C の処理をしてください。
```

**出力フォーマット**:
```bash
# 構造化データ出力
claude -p "全 API エンドポイントをリストアップ" --output-format json

# ストリーミング JSON（リアルタイム処理に）
claude -p "このログを解析" --output-format stream-json | jq '.content'

# デバッグ時のみ詳細出力
claude -p "..." --verbose
```

**ファンアウトパターン（大規模マイグレーション）**:
```bash
# 1. 対象ファイルをリストアップ
claude -p "React を使っている全 .tsx ファイルをリストし、ファイル名だけ出力" > files.txt

# 2. 並列処理
cat files.txt | xargs -P 5 -I{} claude -p \
  "{} を React から Vue に移行してください。OK または FAIL を返してください。" \
  --allowedTools "Edit,Bash(git commit *)"
```

**適用提案**: CI でのレビュー自動化に `--output-format json` + `jq` を組み合わせる。`--allowedTools` でバッチ処理時の権限を最小化する。

### Ctrl+G でプランをエディタで編集

**発見内容**: Plan Mode 中に `Ctrl+G` を押すと、作成されたプランがテキストエディタで開く。Claude が実装に進む前に直接編集できる。

**適用提案**: 4フェーズワークフロー（Explore→Plan→Implement→Commit）の Plan ステップで `Ctrl+G` を活用し、プランを精緻化してから実装に移る。特に複数ファイルにまたがる変更や設計判断が必要な場合に有効。

### よくある失敗パターン（公式アンチパターン）

**発見内容**: Anthropic 公式ドキュメントに 5 つの典型的失敗パターンが明記。

| パターン | 症状 | 対処 |
|---------|------|------|
| **キッチンシンクセッション** | 無関係なタスクが混在、コンテキスト汚染 | タスク切り替えごとに `/clear` |
| **過剰修正ループ** | 同じ間違いを 2 回以上修正しても直らない | 2 回失敗したら `/clear` してプロンプトを書き直す |
| **CLAUDE.md 肥大化** | ルールが埋もれて無視される | 「これがなければ Claude は間違えるか？」で各行を評価 |
| **確認なし信頼** | 見た目は正しいが edge case を処理していない | テスト・スクリプト・スクリーンショットで必ず検証 |
| **無限探索** | 調査スコープが広がりコンテキストを浪費 | 調査範囲を限定するか、サブエージェントに委任 |

**適用提案**: CLAUDE.md に以下を追加。
```markdown
# コンパクション指示
コンパクション時は変更ファイルリストとテストコマンドを必ず保持。
```

### MCP スコープと認証の最新ベストプラクティス

**発見内容**: MCP サーバーのスコープ管理と認証のガイドラインが明確化。コンテキスト削減が 85〜95% に拡充（前回レポートは 85%）。

**3 段階スコープ**:
```bash
# グローバル（全プロジェクトで利用）
claude mcp add --scope user github-mcp

# プロジェクト個人設定（.gitignore 推奨）
claude mcp add --scope local my-db-server

# チーム共有（.mcp.json として git 管理）
claude mcp add --scope project shared-tool
```

**よく使われるサーバー**:
- `github` + `filesystem`: 最高頻度カバー（まず導入）
- `postgresql`: DB クエリが多い場合
- `context7`: ドキュメント参照が多い場合

**適用提案**: 頻繁に使う URL ドメインは `/permissions` で allowlist 登録しておく。
```bash
/permissions
# → allowedDomains に追加
```
